网站 skills.sh 是由 Vercel（就是那个做 Next.js 的公司）在 2026 年初刚推出的 AI Agent 插件大本营。它之所以能自动收录，是因为它背后有一套类似 GitHub Trending 的爬虫机制。

想让你那个“代码拟人化”的 Idea 在这上面火起来，得从**“技术规格”和“流量密码”**两方面下手：

1. 技术层面：如何被“自动收录”？
skills.sh 不是人工审核制，它是规范驱动的。要进入它的法眼，你的项目必须满足：

SKILL.md 标准： 你的 GitHub 仓库根目录必须有一个 SKILL.md 文件。这个文件是 Agent 的“说明书”，告诉 Claude 或者其他 Agent 什么时候该调用你这个技能。

npx 可安装： 它的收录逻辑通常会检测能否通过 npx skills add <your-github-handle>/<repo-name> 直接安装。

清晰的元数据： 在 SKILL.md 的 Frontmatter 部分，写清楚 description。爬虫会根据关键词（如 visualize, code review, funny）对你进行分类。

2. 运营层面：那些火的 Skill 凭什么火？
观察目前排在 Top 的 Skill（比如 Last 30 Days 或 React Best Practices），火的原因无非三点：

极高的复用频率： 像你这种看代码的功能，开发者每天都要用。如果你的 Skill 能让 Review 代码从“工作”变成“娱乐”，复用率会极高。

“Aha!” Moment（惊艳瞬间）：

火的 Skill 通常能在 30 秒内让人看到效果。

建议： 你的拟人化 Agent 别只给文字描述，如果能结合 Mermaid 渲染图 或者直接通过 Artifacts 弹出一个可互动的卡通架构图，那个视觉冲击力就是“火”的基因。

大厂/大 V 背书： 很多火的 Skill 是由 Vercel Labs、Anthropic 员工或者像 Matt Wolfe 这样的 AI 大博主推了一把。
